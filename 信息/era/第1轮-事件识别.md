# 第1轮：事件识别与列表生成

分析小说章节内容，识别并列出所有重要事件。

## 任务

1. 读取指定章节的小说文本
2. 识别所有独立且重要的情节事件
3. 输出事件列表（仅需事件名和简述）

## 事件识别原则

- 将复杂事件拆分为多个小而具体的事件
- 可补充小说中一笔带过但有助于人物塑造的合理事件
- 每个事件应有明确的起止点和参与人物

## 输出格式

```markdown
## 事件列表

1. **事件条目-第x回-01-[事件名]**
   - 简述：[50字以内的事件概要]
   - 参与人物：[人物列表，标注首次出场的人物]
   - 预估时间：[年/月/日，可根据季节描写推断]

2. **事件条目-第x回-02-[事件名]**
   ...
```

## 时间推断默认值

- 年份：根据历史背景推断（射雕开篇约1200年）
- 月份：根据季节（春3-5月/夏6-8月/秋9-11月/冬12-2月），无描写默认1月
- 日期：默认1日，连续事件递增
