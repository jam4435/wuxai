# 第2轮：事件详情与人物差分生成

根据第1轮识别的事件列表，为每个事件生成完整的 JSON 文件，并更新人物变量。

## 任务

1. 读取第1轮生成的事件列表
2. 为每个事件生成完整的事件 JSON
3. 更新相关人物变量文件

## 事件 JSON 格式

```json
{
  "事件地点": "大宋/临安府/牛家村",
  "触发条件": { "类型": "时间", "年": 1200, "月": 1, "日": 16 },
  "事件结束时间": { "年": 1200, "月": 1, "日": 16 },
  "事件详情": "300字左右的详尽叙述...",
  "insert": { "新人物/新属性": {} },
  "update": { "已有人物": { "变化的属性": "新值" } },
  "delete": { "人物": { "要删除的键": {} } }
}
```

## 差分操作规则

| 操作   | 用途                 | 示例                   |
| ------ | -------------------- | ---------------------- |
| insert | 新增不存在的键       | 首次出场人物、新获武功 |
| update | 修改已存在键的值     | 境界提升、性格变化     |
| delete | 删除键（用`{}`标记） | 失去身份、丢失物品     |

**键名替换 = delete旧键 + insert新键**（不能用update）

## 人物变量字段

新人物必须包含：姓名、性别、外貌、出生年份、身材、性格、所在位置、身份、武功、重要物品、人物经历、关系网

## 工作流程

1. 查阅 `射雕/人物变量/` 中已有的人物 JSON
2. 根据人物当前状态决定使用 insert/update/delete
3. 生成事件 JSON 到 `射雕/事件/`
4. 更新人物 JSON 到 `射雕/人物变量/`
